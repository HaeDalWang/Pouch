# 🦦 "내 주머니를 최적화" — 무엇이 철학에 맞나

> **이 문서는 "pouch를 써서 내 하네스·스킬·도구를 최적화한다"는 아이디어가 PM
> 철학의 어디에 맞고 어디서 벗어나는지 가른다.** 그리고 철학 안에서 지을 수 있는
> 후보를 펼쳐 첫 걸음을 고르기 위한 것이다. 아무 기능 아이디어나 여기 나침반에
> 대보면 "이게 pouch다운가"를 스스로 판정할 수 있다.

---

## 나침반 — 어떤 "최적화 기능"이 pouch답나

세 필터를 **다** 통과하면 맞다. 하나라도 걸리면 pouch의 자리가 아니다.

| # | 필터 | 통과 | 탈락 |
|---|---|---|---|
| 1 | **환경 vs 성능** | 무엇이 손에 있고 나에게 맞느냐(환경) | 에이전트가 얼마나 잘·빨리 하느냐(성능) → 다른 하네스 몫 |
| 2 | **제안 vs 강제** | 제안만, 동의 후 실행 | 자동으로 바꿔버림 |
| 3 | **실사용 vs 지어내기** | 실제 사용 신호에서만 | 임의 큐레이션("DevOps면 이거 쓰겠지") |

**추가 코드 원칙:** `vendored`(upstream 있는) 도구의 **본문은 불변** — 개인화는 overlay에.

> 근거: README "pouch가 맡지 않는 자리"(성능 아님·앞단), evolve "제안만",
> BACKLOG 런치템플릿 철회(지어내기 금지), D1(비강제), 🔧 vendored body 불변.

## 네 갈래 판정

| 갈래 | 판정 | 철학 안에서는 |
|---|---|---|
| **구성**(무엇을 손에) | ✅ 정통 | 이미 `evolve`. 더는 리포트·세트 내보내기로 확장 |
| **개인화**(나에게 맞게) | ✅ 정통 | overlay(태그·경계·메모) + 실사용에서 학습(ROADMAP Phase 4.5) |
| **실행 성능**(더 잘·빨리) | ❌ 밖 | pouch 직접 안 함. 환경을 잘 갖춰 *간접적으로* 결과가 나아지는 게 몫. 직접 튜닝은 실행 하네스 |
| **스킬 내용 개선** | ⚠️ 소유권으로 갈림 | `owned`: 개선 **제안** OK. `vendored`: 본문 대신 overlay에 개인화 |

## 철학 안의 후보 (씨앗은 이미 BACKLOG/ROADMAP에 있음)

셋 다 "환경을 실사용 기반으로 제안해 최적화"라 나침반 3필터를 통과한다.

### A. 주머니 리포트 — 기간별 가시성 🌊
"이번 주/달: 뭘 많이 썼나 · 뭐가 안 쓰여 닳는가 · 주머니 밖에서 뭘 자주 쓰나."
- **read-only**(아무것도 안 바꿈) → 위험 0, 3필터 자동 통과.
- `status`의 시간축 확장. 최적화를 *보이게* 만들어 사용자가 evolve로 행동.
- 크기 **작음**. BACKLOG "주간 리포트".

### B. 개인화 학습 — 실사용에서 프로필·태그를 배워 제안 🪨
"너 이 도구 항상 이런 맥락에서 쓰네 — 태그/경계로 굳힐까?"
- 지금 overlay는 손으로 채운다. 이걸 **실사용이 채우게** = "쓸수록 나에게 맞게 닳는다"의 핵심.
- 제안만(잔소리 방지는 이미 있는 nudge 정책 재사용). ROADMAP **Phase 4.5 태그 승격**.
- 크기 **중간**(신호 설계·민감정보 주의).

### C. owned 도구 다듬기 제안
"이 명령 자주 쓰는데 이렇게 바꾸면 어때?"(적용은 사람). vendored는 overlay로 우회.
- 크기 **중간**. "스킬 내용 개선" 욕구를 불변 원칙 안에서.

---

## 밖에 두는 것 (명시)

- **실행 성능 튜닝** — 실행 하네스의 일. pouch는 앞단(환경·기억). 필요하면 pouch가 그
  하네스와 *연동*할 순 있어도, *겸하지*는 않는다.
- **vendored 본문 자동 수정** — 🔧 불변 원칙 위반. 개인화는 언제나 overlay.

## 첫 걸음 — A(주머니 리포트) 완료

`pouch report [--days N]` 냈다(`report.py` + cli.py). read-only로 나침반 3필터를 무조건
통과하고, `status`의 usage 집계 코어를 재사용해 새 시스템을 안 지었다. 세 구역을 보인다:

- **많이 쓴 것** — 기간 내 카탈로그 도구 상위.
- **안 쓰여 닳는 중** — 표면에 있는데 기간 내 안 쓰인 **신호형** 도구(훅·규칙·에이전트는
  신호가 안 찍혀 제외 — evolve의 drop 방어와 같은 정신). → `pouch evolve`로 정리.
- **주머니 밖에서 쓰는 것** — 카탈로그 밖인데 쓴 것. → `pouch evolve`로 편입.

최적화를 *보이게* 만들어 사용자가 evolve로 행동하게 한다(결정은 evolve가, 리포트는 눈).
