# 🦦 맥락 개인화 (레인 2) — 프라이버시 설계 먼저

> **이 문서는 [PERSONALIZATION-LEARNING.md](PERSONALIZATION-LEARNING.md)의 레인 2를 연다.**
> 레인 2는 새 신호(맥락)를 잡는데 그게 민감정보라, **코드 전에 "무엇을 어디까지 잡고
> 어디 두나"의 프라이버시 결정을 먼저 락한다.** 정해지면 근거와 함께 여기 남긴다.

---

## 레인 2가 원하는 것

"어느 프로젝트/작업에서 뭘 쓰나"를 배워 **프로젝트별 태그·경계·구성**을 제안한다.
"쓸수록 나에게 맞춰진다"의 진짜 핵심 — 레인 1(핵심 도구)이 "무엇이 손에 맞나"였다면,
레인 2는 "**어디서** 무엇이 맞나"다.

## 크럭스 — 새 신호는 민감도가 갈린다

현재 usage = **(entry_id, ts)**. 맥락을 잡으려면 신호를 늘려야 하는데, 두 종류다:

| 신호 | 민감도 | 무엇 |
|---|---|---|
| **2a 프로젝트 맥락** | 낮음 | "repo P에서 X를 쓴다" — 경로·프로젝트명. **내용은 아님** |
| **2b 작업/주제 맥락** | 높음 | "무엇을 하려 했나" — 질문·작업. **날것은 절대 금지** |

## 프라이버시 필터 (나침반의 4번째)

기존 3필터(환경·제안·실사용)에 하나 더: **민감 신호는 날것 저장 금지 · 로컬 전용 ·
백업/공유에서 제외.** pouch의 기존 원칙과 정렬 — *"프로젝트별 `.pouch/`는 민감정보가
섞일 수 있어 클라우드에 안 보낸다"*(HOW-IT-WORKS 백업 섹션).

## 2a — 프로젝트 맥락: 어디에 저장하나 (핵심 결정)

usage에 프로젝트를 붙이면 **전역 로그(`~/.pouch/usage.jsonl`, 백업에 포함)**에 경로가
들어간다 — 경로가 클라이언트명 같은 걸 흘릴 수 있다. 세 선택:

| | 방식 | 평가 |
|---|---|---|
| P1 | 원경로를 전역 로그에 | 간단하나 **백업/공유에 경로 노출** — 비추 |
| P2 | 해시 + 로컬 전용 이름맵 | 백업엔 해시만, 이름 해석은 로컬 파일(백업 제외) |
| **P3** | **프로젝트 `.pouch/`에 per-project 관찰** | 프로젝트 `.pouch/`는 이미 로컬 전용·백업 제외라 원칙과 정렬. 전역 로그는 그대로(맥락 없음) |

→ **추천 P3.** 원경로를 전역 로그에 안 넣는다. 훅이 로깅할 때 현재 프로젝트의
`.pouch/usage.jsonl`(로컬 전용)에도 한 줄 남기고, `pouch report`를 프로젝트에서 돌리면
그 로컬 로그로 "이 프로젝트에서 뭘 쓰나"를 보여준다.

## 2b — 작업/주제 맥락: 더 뒤로

주제 추출·동의·수명 설계가 선행돼야 한다(BACKLOG "질문·작업 반복 신호원"이 못박은
*주제·꼬리표만, 날것 금지*). **2a 다음** — 이번엔 열지 않는다.

## 첫 걸음 — 2a P3 읽기전용 가시성 완료

- **신호(P3)** — `pouch evolve log`(PostToolUse 훅)가 전역 로그에 더해, 프로젝트 안이면
  그 repo의 `.pouch/usage.jsonl`(로컬 전용)에도 이중 기록한다(`paths.project_usage_log_path`).
  프로젝트 경로·맥락은 전역 백업에 안 들어간다. `.gitignore`에 `.pouch/` 추가로 커밋도 차단.
- **가시성** — `pouch report`를 프로젝트에서 돌리면 "📁 <repo>에서 많이 쓴 것 (로컬 전용)"
  구역이 뜬다. 전역 로그는 그대로(맥락 없음) — 나침반 4필터 통과(환경·제안·실사용·로컬전용).

**밖에 두는 것(다음):**
- **프로젝트별 주머니**(per-project 카탈로그/표면) — 프로젝트-스코프 *제안*의 진짜 payoff지만
  카탈로그 모델 확장이라(BACKLOG "프로젝트별 주머니") 별도. 이번 가시성이 그 토대(신호)를 놓았다.
- **2b 작업/주제** — 프라이버시 설계(주제·꼬리표만, 날것 금지) 선행.
